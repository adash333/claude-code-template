## やり取り 3: Stop フックの動作確認

### ユーザー

> 会話ログの自動保存先（フックにより生成）は、毎回は発動しないのですか？

### アシスタント（要約）

[.claude/hooks/save_prompt_log.py](../../.claude/hooks/save_prompt_log.py) と [.claude/settings.json](../../.claude/settings.json) を確認した結果:

- Stop フックは matcher: `""` で登録されており、Claude の応答完了のたびに **毎回発動する**
- ただしスクリプト側で以下のケースはスキップ（記録しない）:
  - `<system-reminder>` タグだけ／空白だけのプロンプト（`_is_only_system_noise`、[save_prompt_log.py:41](../../.claude/hooks/save_prompt_log.py#L41)）
  - transcript が読めない／パースできないケース
  - 直近の user メッセージが取れないケース
  - ファイル書き込み失敗時（`exit 0` で静かに抜けて Claude を妨げない）
- 通常のユーザープロンプト + 応答は毎回 `docs/prompt/YYYY-MM-DD.md` に追記される
