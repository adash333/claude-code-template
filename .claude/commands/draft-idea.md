# アイデアメモの作成（対話形式）

このコマンドは、Claude が段階的に質問を投げかけ、その回答をもとに `docs/ideas/initial-requirements.md` を自動生成します。テンプレートのプレースホルダー（`{...}`）を手作業で埋める代わりに、対話で必要な情報を引き出します。

## 実行方法

```bash
claude
> /draft-idea
```

## 手順

1. **draft-idea スキル**をロードする
2. スキルの「実行手順」に従って対話を開始する:
   - Phase 0: 既存ファイルの確認（編集済みなら上書き可否を確認）
   - Phase 1（必須）: プロダクトの核（名前・キャッチコピー・概要）
   - Phase 2（必須）: 解決したい課題
   - Phase 3（必須）: ターゲットユーザー
   - Phase 4（必須）: MVP の機能候補
   - Phase 5（任意）: 差別化ポイント
   - Phase 6（任意）: 技術的な検討事項
   - Phase 7（任意）: 非機能要件・成功指標
3. すべての回答が揃ったら、テンプレートの構造に従って `docs/ideas/initial-requirements.md` を生成する
4. 生成完了をユーザーに報告し、次のステップ（`/setup-project`）を案内する

## 注意事項

- 質問は **Phase 単位** で進めます（一度に全部聞きません）
- 任意フェーズは「スキップ」「わからない」と答えれば未入力のままにできます
- 既に編集済みの `initial-requirements.md` がある場合、上書き前に確認します
