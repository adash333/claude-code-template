# Save Prompt Log スキル

ユーザーとのやり取りを `docs/prompt/` 配下に Markdown ファイルとして保存し、後から振り返り可能な会話ログを蓄積するスキル。

## スキルの目的

- 重要な意思決定や相談プロセスを文書として残す
- スペック駆動開発の文脈で「なぜこの判断をしたか」を記録する
- 過去のやり取りを後から検索・参照可能にする

## 使用タイミング

以下のいずれかに該当するとき:

1. **ユーザーから明示的に「このやり取りを保存して」と指示されたとき**
2. **重要な意思決定・方針合意があったとき**(機能の選定、設計方針の決定、トレードオフ判断など)
3. **CLAUDE.md やドキュメントへ反映する前段階の議論を残したいとき**

雑談・単純な質問への即答などは記録しなくてよい。

## ファイル命名規則

```
docs/prompt/YYYY-MM-DD-{トピック}.md
```

- `YYYY-MM-DD`: 会話開始日(currentDate を使用)
- `{トピック}`: 内容を端的に表す日本語スラッグ(例: `開発フロー相談`、`認証機能設計`)

**同日に同トピックの追加やり取りが発生した場合**: 既存ファイルに追記する(新規作成しない)。
**同日に別トピックの会話が発生した場合**: 別ファイルとして新規作成する。

## ファイルフォーマット

```markdown
---
日付: YYYY-MM-DD
トピック: {1行説明}
---

# YYYY-MM-DD {トピック}

## やり取り 1: {このやり取りの主題}

### ユーザー

> {ユーザーのプロンプト全文(短ければ全文、長ければ要約 + 重要部分の引用)}

### アシスタント(要約)

{応答の要点を箇条書き or 段落で要約。コードや長い表は省略可、ただしファイルパス・決定事項は残す}

---

## やり取り 2: {次の主題}

(同じ構造で繰り返し)

---

## 決定事項・ToDo

- [ ] {次回以降に持ち越す項目}
- [x] {このセッションで合意した項目}
```

## 実行手順

### 手順 1: 既存ファイルの確認

```bash
ls docs/prompt/
```

`docs/prompt/` が存在しない場合は Write 実行時に自動作成される。

### 手順 2: トピックの判定

- 直前のファイルと同一トピックの継続か?
- 同一なら追記、異なるなら新規作成

### 手順 3: 内容の整形

- ユーザープロンプトは原則そのまま引用(ブロッククォート)
- アシスタント応答は **要約** する(全文をダンプしない)
- 決定事項・採用された選択肢は明示
- ファイルパス・コマンドは省略せず残す

### 手順 4: 保存

- 新規: `Write` ツール
- 追記: `Read` で末尾確認 → `Edit` で `---` の前に挿入、または `Write` で全体書き換え

### 手順 5: 確認

保存後、ユーザーに「`docs/prompt/YYYY-MM-DD-{トピック}.md` に保存しました」と1行で報告。

## 注意事項

- **シークレット情報は記録しない**: API キー、認証情報、個人の取引データなどが会話に含まれる場合は伏字化(`****`)してから保存
- **長すぎる応答は要約**: コードブロック全文や長大な表はリンクで参照する形にする
- **記録しすぎない**: 雑談や単純な確認応答は記録不要。重要な判断や設計議論に絞る

## 自動実行について

このスキルは Claude が状況判断で呼び出すもので、**全プロンプトに自動実行されるわけではない**。
すべてのやり取りを機械的に記録したい場合は、`UserPromptSubmit` フック または `Stop` フックを `.claude/settings.json` に設定する必要がある(`update-config` スキル参照)。
